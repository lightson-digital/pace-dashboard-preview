# Pace Dashboard v5 — Revised Design Spec

**Status:** Draft
**Date:** 2026-04-07
**Revision of:** `pace-pipeline/docs/superpowers/specs/2026-03-15-pace-dashboard-design.md` (v4, approved 2026-03-15)
**Trigger:** Team feedback via Loom video review of v4 mockup

---

## Purpose

A standardized weekly pace report for LOD hotel clients. Replaces manual Excel reports with an automated HTML dashboard for both LOD team (interactive browser) and hotel owners (PDF export).

**Design philosophy:** Think like a hotel management company, not an agency. Hoteliers think in month boundaries and full-year targets. Simplify — subtract, don't add.

---

## What Changed from v4

| v4 | v5 | Why |
|----|-----|-----|
| 90-day rolling window as primary frame | Rolling 12 months | Hoteliers think in months and full-year targets, not arbitrary 90-day windows |
| 90-day aggregate KPI banner | Rolling 12-month KPI banner | "How ahead/behind for the year?" is the real question |
| Calendar month tiles (3-4 months) | Removed | Redundant with monthly breakdown table |
| Separate occupancy line chart | Removed — occupancy lives in the hero chart and table | Simplify |
| Separate pickup bar chart | Removed — pickup data lives in the monthly/daily breakdown | Simplify |
| No pickup ADR | Pickup ADR column added | Quick anomaly indicator for rate quality of new bookings |
| No budget variance | Budget column designed, shipped disabled | Schema ready, no data yet |
| No time filter on charts | Hero chart has 3/6/9/12 month filter (default 12) | Zoom into near-term or see full year |

---

## Architecture (unchanged from v4)

- Single HTML file per hotel per snapshot, generated by Python + Jinja2
- All data pre-computed and embedded as JSON in the HTML
- Vanilla JS for interactivity (comparison toggle, OOO toggle, chart filter, expand/collapse)
- Inline SVG for charts — no external chart libraries
- `@media print` hides browser controls, optimizes for letter-size PDF
- Single `<style>` block, no external CSS
- No external dependencies

## Page Structure

Top-to-bottom layout, each section described below:

### 1. Report Header

Black background, LOD branding. Hotel name, report title ("Pace Report"), snapshot date prominently displayed.

Same as v4. No changes.

### 2. Browser Controls Bar (interactive only, hidden in print)

Light gray bar containing:
- **Hotel selector** — Dropdown. Navigates to another hotel's report file. Placeholder in V1 (single-hotel file).
- **Comparison toggle** — Pill buttons: `WoW` | `STLY` | `Budget`. Budget is visually present but grayed out with "(no data)" tooltip. Default: STLY.
- **OOO toggle** — Pill buttons: `Incl OOO` | `Excl OOO`. Default per hotel config (`occ_excludes_ooo` flag).
- **Snapshot date display** — Read-only. Shows the snapshot date the report was generated from.
- **Export PDF** — Button. Triggers `window.print()`.

### 3. Rolling 12-Month KPI Banner

Aggregate KPIs for the rolling 12-month window (current month + 11 months forward).

**Layout:** Single row of 4 metric cards, each showing:
- Metric label
- Current value (large)
- Delta vs comparison (arrow + percentage + absolute value)
- Color-coded: green positive, red negative

**Metrics:**
1. **OTB Revenue** — Sum of `room_revenue` across the 12-month window
2. **Blended ADR** — Total room revenue / total rooms sold
3. **Avg Occupancy** — Total rooms sold / total available room-nights (adjusted for OOO per toggle)
4. **RevPAR** — Total room revenue / total available room-nights (adjusted for OOO per toggle)

**Comparison:** Deltas are calculated against the active comparison mode (WoW or STLY). The comparison snapshot's 12-month window uses the same stay-date range but from the comparison snapshot date.

### 4. Hero Pacing Chart

**Type:** Combination chart — grouped bars + line on secondary axis.

**Bars (primary Y-axis):**
- This year OTB revenue (solid bar, electric blue `#0A84FF`)
- Comparison revenue (outline/ghost bar, light gray `#C0C0C0`)
- Grouped by month (one group per calendar month)

**Line (secondary Y-axis):**
- Blended ADR for each month (this year)
- Thin line with dot markers, dark color (`#1A1A1A`)

**X-axis:** Calendar months. Labels as "Mar", "Apr", etc.

**Time filter pills:** `3` | `6` | `9` | `12` — controls how many months the chart displays. Default: `12`. Pills are positioned top-right of the chart area. This filter controls ONLY the chart; the monthly breakdown table always shows rolling 12 months.

**Bar granularity:** Monthly. One bar group per calendar month.

**Hover/interaction:** None in V1. Static SVG.

**Print:** Chart renders at full width, time filter pills hidden. Always prints 12-month view.

### 5. Monthly Breakdown Table

Rolling 12 months from current month. Always shows all 12 months regardless of hero chart filter.

**Columns:**

| Column | Description |
|--------|-------------|
| Month | Calendar month label (e.g., "Apr 2026") |
| Rooms | Rooms sold (OTB) |
| Rooms Δ | Delta vs comparison |
| Revenue | Room revenue (OTB) |
| Revenue Δ | Delta vs comparison |
| ADR | Blended ADR (room revenue / rooms sold) |
| ADR Δ | Delta vs comparison |
| Occ% | Occupancy percentage (adjusted per OOO toggle) |
| Occ Δ | Delta vs comparison (in percentage points) |
| RevPAR | Revenue per available room (adjusted per OOO toggle) |
| RevPAR Δ | Delta vs comparison |
| Pickup Rooms | Room pickup since comparison snapshot |
| Pickup Rev | Revenue pickup since comparison snapshot |
| Pickup ADR | Pickup revenue / pickup rooms. Anomaly indicator. |

**Delta formatting:**
- Positive: green text, `+` prefix, up arrow
- Negative: red text, `−` prefix, down arrow
- Zero/negligible: neutral gray

**Row expansion:** Each month row is expandable (click/tap) to show daily breakdown. Expansion state is browser-only; print always shows collapsed.

**Budget column (future):** When budget data is available, add `Budget` and `Budget Δ` columns after Revenue Δ. These show the budget target and the gap ("the chase"). Disabled in V1 — columns not rendered until data exists.

### 6. Daily Breakdown (expanded view)

Nested table within each month row. Shows per-day detail for that month.

**Columns:** Date | Rooms | Revenue | ADR | Occ% | RevPAR | Pickup Rooms | Pickup Rev | Pickup ADR

All values show current OTB. Delta vs comparison shown inline (parenthetical or secondary line) for Rooms, Revenue, ADR.

**Browser only.** Not rendered in print/PDF to keep report concise.

### 7. Footnotes

- OOO adjustment explanation: "Occupancy and RevPAR calculated excluding X out-of-order rooms" (or "including" per toggle)
- ADR definition: "ADR = Room Revenue / Rooms Sold (excludes ancillary revenue where available)"
- Comparison mode explanation: "Compared to snapshot from [date] ([mode label])"
- Data freshness: "Generated from [snapshot_date] snapshot"

### 8. Footer

Black background, LOD branding, generation timestamp, "Confidential — prepared for [Hotel Name]".

Same as v4. No changes.

## Comparison Modes

| Mode | Label | Logic | Status |
|------|-------|-------|--------|
| WoW | Week over Week | Compare to snapshot from 7 days prior. Fallback: ±1 day. | Active |
| STLY | Same Time Last Year | Compare to snapshot from 364 days prior (52 weeks). Fallback: ±3 days. | Active (default) |
| Budget | Budget | Compare OTB to budget snapshot for same stay dates. | Disabled — no data. Pill visible but grayed out. |

LYA (Last Year Actual) removed from v5 toggle — requires actuals ingestion. Will return as part of the calendar-year view follow-up.

## Occupancy Handling

Two calculation modes:
- **Excl OOO:** denominator = `available_rooms - ooo_rooms` per day
- **Incl OOO:** denominator = `available_rooms` per day (= `hotel.total_rooms`)

Toggle in browser controls. Default per hotel config (`hotels.occ_excludes_ooo`).

**PDF export honors the current OOO selection**, not the hotel default. The printed footnote states which mode was used: "Occupancy and RevPAR calculated excluding/including out-of-order rooms." This ensures the PDF matches what the user reviewed on screen before exporting.

Both occupancy variants are pre-computed and embedded in the JSON payload. Toggle swaps which dataset is displayed — no recalculation in JS.

### Zero-Availability Handling

When `available_rooms - ooo_rooms = 0` for a day (full renovation, closure), or when `rooms_sold = 0` for a period:

**Pipeline (generate.py):**
- Occupancy and RevPAR: emit `null` in the JSON payload when denominator is zero
- ADR: emit `null` when `rooms_sold = 0`
- Pickup ADR: emit `null` when `pickup_rooms = 0`
- Deltas: suppress (emit `null`) when either current or comparison value is `null`

**Renderer (JS/template):**
- `null` values display as "—" (em dash) in all contexts (KPI cards, table cells, chart labels)
- Delta arrows and color coding are suppressed for `null` values — show neutral gray "—"
- KPI banner aggregates: exclude `null` days from denominator calculations (i.e., a closed day doesn't count toward available room-nights)
- Hero chart: bars for months with zero revenue show as empty (no bar), ADR line skips `null` points

This applies at every aggregation level: daily, monthly, and 12-month KPI banner.

All comparison modes are pre-computed server-side and embedded. JS toggles between `modes.stly`, `modes.wow`, etc. — no client-side calculation.

### Unavailable Comparison Mode Contract

When a comparison snapshot cannot be found (fallback exhausted), the mode object is set to `null` in the payload. Example: `"budget": null`, or `"stly": null` if no snapshot exists 364±3 days prior.

**UI behavior when a mode is `null`:**
- **Comparison pill:** Grayed out with "(no data)" tooltip. Not selectable.
- **KPI banner:** If the *active* mode becomes unavailable (shouldn't happen since pills prevent selection), fall back to showing current values only with "No comparison" label and no deltas.
- **Hero chart:** Only current-year bars render. Ghost comparison bars are omitted. ADR line still shows.
- **Monthly/daily tables:** Delta columns show "—" throughout. Pickup columns show "—" (pickup requires two snapshots).

**Edge case — all modes unavailable:** Dashboard renders in "current-only" mode. KPI banner shows raw values with no deltas. Chart shows only current bars. Tables show current values with all delta/pickup columns as "—". A banner message appears: "No comparison data available for this snapshot."

The JS rendering functions check `DATA.modes[activeMode] === null` before accessing any nested properties.

## Print / PDF Layout

- **Page size:** US Letter, margins 0.35in sides / 0.45in top-bottom
- **Page 1:** Header + KPI banner + hero chart (12-month, no filter pills) + top of monthly table
- **Page 2+:** Rest of monthly table + footnotes + footer
- **OOO mode:** Print honors the user's current OOO toggle selection (not the hotel default)
- **Hidden in print:** Browser controls bar, time filter pills, daily breakdown, expand/collapse controls
- **Font sizes:** Reduced 20-30% from browser view
- **Colors:** Preserved exactly (`-webkit-print-color-adjust: exact`)

## Non-Goals (out of scope for v5)

1. **Portfolio view** — Multi-hotel summary page. Comes after single-hotel is standardized.
2. **Pickup by segment** — Not available in current data model. Future PMS integration work.
3. **Price report** — Lighthouse covers this. Only one team member uses it.
4. **Calendar year view** — Requires actuals ingestion. Fast follow-up once `snapshot_type='actual'` data lands.
5. **Last Year Actual comparison mode** — Same blocker as calendar year view.
6. **Interactive chart hover/tooltips** — Keep V1 static. Add interactivity in a later pass.
7. **Real-time data** — Dashboard is generated from snapshots, not live queries.
8. **Multi-hotel file navigation** — Hotel selector is a placeholder in V1 (single-hotel files).

## Acceptance Criteria

### KPI Banner
- **Given** a snapshot for KAH dated 2026-04-05 with STLY comparison, **when** the dashboard loads, **then** the KPI banner shows rolling 12-month aggregates (Apr 2026 – Mar 2027) with deltas vs the STLY snapshot's same stay-date range.
- **Given** the user toggles OOO to "Incl OOO", **then** Occupancy and RevPAR KPIs update to use `incl_ooo` values. Revenue and ADR remain unchanged.

### Hero Chart
- **Given** the dashboard loads, **then** the hero chart shows 12 monthly bar groups (default) with revenue bars and ADR line.
- **Given** the user clicks the "3" time filter pill, **then** the chart re-renders showing only 3 months of bars. The monthly breakdown table below is unaffected.
- **Given** the user switches comparison to WoW, **then** the ghost bars update to show WoW comparison revenue. The chart re-renders.
- **Given** the dashboard is printed, **then** the chart always shows 12 months. Filter pills are not visible.

### Monthly Breakdown
- **Given** the dashboard loads, **then** the table shows 12 rows (one per month, Apr 2026 – Mar 2027) with all columns including Pickup ADR.
- **Given** the user clicks a month row, **then** it expands to show daily breakdown for that month.
- **Given** a month where pickup rooms = 0, **then** Pickup ADR shows "—" (not division by zero).
- **Given** comparison mode is Budget and no budget data exists, **then** budget columns are not rendered.

### Comparison Modes
- **Given** the user clicks "Budget" pill, **then** nothing happens (pill is grayed out, shows tooltip "No data available").
- **Given** the user switches from STLY to WoW, **then** all KPIs, chart bars, table deltas, and pickup values update to reflect WoW comparison.

### Accuracy
- Revenue values accurate to ±$0.01 vs source query.
- Occupancy percentages accurate to ±0.01pp vs source query.
- ADR values accurate to ±$0.01 vs source query.
- Pickup ADR = pickup revenue / pickup rooms, no rounding until display (2 decimal places).

### Zero-Availability
- **Given** a day where all rooms are OOO (denominator = 0) in Excl OOO mode, **then** Occ% and RevPAR show "—" for that day. The month aggregate excludes that day from its denominator.
- **Given** a month where rooms sold = 0, **then** ADR shows "—" for that month. KPI banner ADR excludes that month from the weighted average.

### Missing Comparison Data
- **Given** STLY comparison snapshot not found (fallback exhausted), **then** `modes.stly` is `null`. The STLY pill is grayed out. If STLY was the default, fall back to WoW. If WoW is also `null`, render in current-only mode.
- **Given** all comparison modes are `null`, **then** dashboard renders current values only. All delta and pickup columns show "—". Banner displays "No comparison data available for this snapshot."

### Print / OOO
- **Given** the user selects "Incl OOO" and clicks Export PDF, **then** the PDF shows Incl OOO values (not the hotel default). The footnote reads "Occupancy and RevPAR calculated including out-of-order rooms."

### Generation
- Dashboard HTML generation completes in <5 seconds per hotel.
- Missing comparison snapshot (no fallback found) → mode set to `null` in payload, dashboard renders gracefully per the unavailable comparison contract.
