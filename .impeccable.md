## Design Context

### Users

**Primary:** Hotel owners and GMs at independent/boutique properties in Hawaii. They receive this as a weekly PDF (or open in browser) to review forward-looking revenue performance. They think in calendar months and full-year targets. They're busy operators, not data analysts — the dashboard needs to surface insights at a glance, not require interpretation.

**Secondary:** LOD team (revenue strategists). They use the interactive browser version with comparison toggles, chart filters, and daily drill-down to prepare talking points for client calls.

**Job to be done:** "Am I ahead or behind on revenue compared to last year / last week, and by how much?" Answered in under 10 seconds of looking at the page.

### Brand Personality

**Lights On Digital:** Confident, minimal, precise.

Visual identity communicates a premium consulting firm that happens to work in hospitality. Like walking into a well-designed office — nothing unnecessary. Credible and calm, not salesy or cluttered.

**Emotional goals:** Confidence (the data is authoritative), calm (clean layout reduces cognitive load), clarity (no ambiguity in what numbers mean).

### Aesthetic Direction

**Theme:** LOD-branded but light. Black header and footer per LOD brand identity. White/light data area for maximum chart and table readability. Brand communicated through typography, color accents, and structural polish — not through dark backgrounds on data-dense sections.

**Typography:** Inter as primary font (Sohne fallback — not embedded due to licensing). Display headings use light weight (300) per LOD brand signature. Section labels: semibold, ALL CAPS, letter-spaced. Body: regular weight, clean reading.

**Color palette:**

| Token | Value | Usage |
|-------|-------|-------|
| `--lod-black` | `#000000` | Header/footer backgrounds, heavy borders |
| `--lod-white` | `#FFFFFF` | Page background, text on dark |
| `--lod-near-black` | `#1A1A1A` | Body text, ADR line |
| `--lod-light-gray` | `#E5E5E5` | Dividers, secondary backgrounds |
| `--lod-electric-blue` | `#0A84FF` | Primary accent — current revenue bars, active pills, interactive elements |
| `--pos` | `#1a8c52` | Positive deltas (muted green, not neon) |
| `--neg` | `#d63b3b` | Negative deltas (muted red, not neon) |
| `--watch` | `#d97706` | Caution state |
| `--ghost` | `#C0C0C0` | Comparison bars (outlined) |

**Anti-references:** Generic Bootstrap dashboards. Neon-on-dark "hacker" aesthetics. Overly decorated admin panels with gradients and shadows.

**References:** Apple's financial reporting PDFs. Stripe's dashboard. The Economist's data pages. Clean, authoritative, restrained.

### Accessibility

**Target:** WCAG AA compliance.
- Text contrast: 4.5:1 minimum (normal text), 3:1 minimum (large text / UI components)
- Color is never the sole indicator — deltas use arrows/symbols alongside color
- Print: `-webkit-print-color-adjust: exact` for faithful PDF rendering

### Design Principles

1. **Data first, decoration never.** Every pixel serves comprehension. No ornamental elements. If it doesn't help the user understand their revenue position faster, remove it.

2. **Scannable at arm's length.** The KPI banner answers the top-level question in under 3 seconds. Monthly table answers "which months?" in 10 seconds. Daily drill-down is opt-in, not mandatory.

3. **Light weights for headlines, heavy weights for data.** LOD's signature: display text is thin/light (300), while metric values are bold (700). This creates visual hierarchy through weight contrast rather than size alone.

4. **Blue as punctuation, not paint.** Electric Blue highlights one focal element per composition — the current revenue bars, the active pill. Everything else is black, white, or gray.

5. **Print is a first-class citizen.** Every design decision must work in both browser and letter-size PDF. If it can't print cleanly, it doesn't ship.
